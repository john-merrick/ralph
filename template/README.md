# Ralph loop — portable starter kit

Drop this folder into a new project to run the autonomous Ralph loop with the
control layer (audit / cost / schedule). Everything here is self-contained and
stdlib-only (Python 3 + bash + `jq`). Defaults to `claude`.

## Files

| File | Role |
|---|---|
| `ralph.sh` | The loop. Runs the agent up to N iterations; stops on `<promise>COMPLETE</promise>`. |
| `CLAUDE.md` | Agent instructions (the prompt `claude` reads each iteration). |
| `prd.json.example` | Work-queue template → copy to `prd.json` and edit. |
| `progress.txt` | Append-only log + Codebase Patterns the agent reads each run. |
| `ralph` | Dispatcher: `ralph audit \| cost \| schedule \| run`. |
| `ralph-run.sh` | Launch wrapper — tags the run, writes a dated rundown. |
| `project_audit.py` | `ralph audit` — morning rundown (progress, blocked, cost). |
| `ralph_cost.py` | `ralph cost` — spend analysis / forecast. |
| `ralph_schedule.py` | `ralph schedule` — opt-in cron install/remove. |
| `ralph_prd.py` | Shared PRD-reading helper (imported by the 3 scripts above). |
| `cost_hook.py` | SessionEnd hook → appends one line to `cost.jsonl`. |
| `.claude/settings.json` | Wires the cost hook. |

Runtime files (`cost.jsonl`, `rundown/`, `archive/`, `.last-branch`,
`__pycache__/`) are generated by the loop and gitignored — don't copy them.

## Setup in a new project

```bash
cp -r template/. /path/to/new-project/    # copy contents into the repo root
cd /path/to/new-project
cp prd.json.example prd.json               # then edit prd.json: branchName + items
./ralph run 10                             # run the loop (tagged + writes rundown)
./ralph audit                              # morning rundown
```

`cost_hook.py` is wired via `$CLAUDE_PROJECT_DIR/cost_hook.py`, so it must sit at
the project root (it does, after the copy above).
